University Dissertation Artifact

PCG Dungeon Generator

Platform: PC

Engine: Unity

Development Time: 6 Months

Role: Solo Developer

Tools: C#, Unity

Project Overview

For my university dissertation, I developed a PC-based procedural content generation (PCG) artifact focused on dungeon generation techniques commonly used within game development. The project explores and critically analyses a range of PCG algorithms, examining how each method impacts spatial structure, readability, and player navigation.

The artifact functions as both a playable prototype and a visual research tool, allowing dungeon layouts to be generated, observed, and compared in real time. Rather than prioritising randomness alone, the project focuses on designer control, consistency, and the practical application of PCG within real game development scenarios.

Gameplay Mechanics

The project demonstrates six of the most commonly used dungeon generation methods within the games industry: Cellular Automata, Binary Space Partitioning (BSP), Perlin Noise, Recursive Division, Drunkard Walk, and Recursive Backtracking.

Each algorithm can be individually selected and visualised, enabling direct comparison of dungeon density, room connectivity, corridor flow, and traversal patterns. Adjustable parameters allow small changes to be made to each algorithm, highlighting how procedural rules influence the final generated space.

Key Responsibilities & Contributions

  • Designed, implemented, and documented multiple procedural dungeon generation algorithms
  • Conducted academic research into procedural level design and spatial generation techniques
  • Developed visual debugging and comparison tools to analyse dungeon readability and flow
  • Created a modular system allowing rapid switching and evaluation of generation methods
  • Tested and refined dungeon layouts to improve player navigation and spatial clarity
